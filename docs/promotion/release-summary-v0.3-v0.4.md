# v0.3 / v0.4 更新摘要

## v0.3：可审查安装计划

v0.3 把 StackPilot 从“只给推荐报告”推进到“生成可审查安装计划”：

- 根据推荐结果生成安装步骤；
- 为步骤补充来源、风险等级、验证命令和回滚命令；
- 生成安装审计报告；
- 生成备份 / 快照计划；
- 生成回滚计划；
- 生成 dry-run 预览；
- 默认不执行任何真实安装。

这一版的核心边界是：先让用户看懂计划，再谈是否执行。

## v0.4：GPU Detection Hardening

v0.4 重点强化 GPU 检测和推荐规则：

- 区分核显、独显、虚拟显卡和未知显卡；
- 增加 `primary_gpu`；
- 增加 `gpu_selection_reason`；
- 增加 `vram_confidence`；
- 避免把共享内存写成独立显存；
- 避免把虚拟显卡当成主要性能 GPU；
- 在 ComfyUI、本地大模型、游戏、创作等场景里给出更保守的 GPU 风险提示。

## 现在需要验证的问题

- 不同 Windows 机器上的 GPU 名称是否能正确分类；
- NVIDIA / AMD / Intel / 虚拟显卡的推荐文案是否清楚；
- 没有 NVIDIA 显卡时，CUDA 相关提示是否足够保守；
- 安装计划里的来源、风险和回滚说明是否便于人工审查。

StackPilot 仍然保持本地优先和 dry-run first。当前版本不会下载软件、不会自动安装，也不会修改系统设置。
